# human2skill 产品规格

## 1. 背景

human2skill 的目标是把有限语料和主动对话转化成可运行的人物视角 Skill。典型语料包括文章、朋友圈、微博、工作文档、聊天记录、会议记录、访谈回答和用户观察。

第一版重点服务私人/身边人。系统不追求“复制一个人”，而是提炼这个人稳定的思维方式、表达习惯、关系互动模式、工作方法和边界。最终产物默认是“视角顾问”，用于帮助用户理解此人可能如何看问题、如何表达、如何给反馈。

## 2. 用户目标

- 在语料有限的情况下快速建立一个有用的人物视角。
- 在无语料或语料薄弱时，通过自适应访谈补足关键信息。
- 能持续补充语料，让人物 Skill 逐步进化。
- 能区分已证实内容、观察者描述和模型推断。
- 能知道哪些结论可信，哪些结论只是低置信度猜测。
- 能安全处理私人语料，不把敏感原文默认打包进可分发 Skill。

## 3. 非目标

- 不做目标人物本人授权平台。
- 不提供伪装成目标人物与第三方互动的能力。
- 不自动代发消息、邮件或社交媒体内容。
- 第一版不做复杂 UI dashboard。
- 第一版不把公众人物深度网络调研作为主路径。
- 第一版不承诺自动解析所有聊天导出格式，只定义可扩展导入策略。

## 4. 核心用户流程

1. 用户启动 `human2skill`。
2. 系统确认人物 profile：同事/合作者、朋友/亲人、导师/专家、自己。
3. 系统确认用途、关系、同意状态、分发状态和原始语料留存偏好。
4. 用户提供已有语料，或直接进入访谈。
5. 系统分析当前信息覆盖度，生成信息缺口。
6. 系统启动自适应访谈，问题根据缺口动态生成。
7. 系统生成私有证据包，区分三类证据和置信度。
8. 系统提炼人物视角，包括思维模型、表达 DNA、profile 专项层、压力反应和边界。
9. 系统生成可运行 `SKILL.md`。
10. 双评审检查证据一致性、表达相似度、边界诚实度和使用价值。
11. 若通过，保存版本；若不通过，进入补充访谈或降低相关结论置信度。

## 5. 人物 Profile

所有 profile 都共享“思维 + 表达”内核，并根据人物身份调整专项权重。

| Profile | 适用对象 | 专项重点 |
| --- | --- | --- |
| 同事/合作者 | 同事、上级、下级、客户、合作伙伴 | 工作方法、职责边界、输出习惯、协作风格 |
| 朋友/亲人 | 朋友、伴侣、父母、子女、亲密关系对象 | 关系互动、情绪反应、支持方式、边界 |
| 导师/专家 | 老师、导师、顾问、创作者、专业人士 | 观点体系、判断框架、反馈方式、表达风格 |
| 自己 | 用户本人、过去的自己、未来理想自我 | 自我反思、决策镜像、长期偏好、盲区 |

## 6. 产物

每个人物生成一个人物目录：

```text
outputs/{slug}/
  person.meta.json
  public_skill/
    SKILL.md
  private_evidence/
    evidence_pack.json
    source_index.json
    interviews/
    reviews/
  versions/
```

`public_skill/SKILL.md` 是可使用、可安装、可分享的轻量产物。默认不包含原始聊天、朋友圈、工作文档全文或其他敏感原文。

`private_evidence/` 是本地私有证据包，用于追溯、评审和后续增量进化。

## 7. 隐私与同意策略

- 默认低摩擦本地自用，不强制要求目标人物本人同意。
- metadata 必须记录：
  - 是否获得本人同意。
  - 是否允许分发。
  - 原始语料是否留存。
  - 用途说明。
- 默认不把原始私域语料写入可分发 Skill。
- 若用户选择留存原文，原文只进入本地私有证据区。
- 输出默认标注为“基于材料生成的视角顾问”，不冒充本人。

## 8. 成功标准

- 用户可以在少量语料或访谈基础上生成一个可运行人物 Skill。
- 生成 Skill 能在新问题上给出有辨识度、可解释、有用的视角。
- 每个核心结论能追溯到证据类型和置信度。
- 双评审能发现证据不足、过度拟合、过度冒充和边界缺失。
- 新语料可以增量合并，并记录版本和置信度变化。
