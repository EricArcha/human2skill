# 自适应访谈设计

## 1. 核心原则

访谈是信息缺口驱动。20 轮是硬限制——达到上限后 Agent 必须暂停由用户决定：

- 信息足够时提前结束。
- 信息不足时继续追问。
- 到 20 轮仍不足时，Agent 暂停并要求用户确认延展或降级。
- 用户确认后继续；用户不想继续时，缺口进入诚实边界，相关结论降置信度。

## 2. 两种访谈视角

### 本人版

目标人物本人回答。权重较高，适合 self profile 或目标人物主动参与的场景。

关注：

- 自我认知。
- 决策偏好。
- 价值排序。
- 语言表达。
- 对自身盲区的承认方式。

### 观察者版

用户作为观察者回答。适合朋友、亲人、同事、导师等身边人场景。

关注：

- 可观察行为。
- 具体场景。
- 原话或近似表达。
- 用户与目标人物的关系位置。
- 观察者偏差。

## 3. 覆盖维度

访谈系统维护一个 coverage map：

| 维度 | 目标 |
| --- | --- |
| 身份与关系语境 | 目标人物是谁，和用户是什么关系 |
| 思维模型 | 他如何判断问题、排序价值、形成观点 |
| 表达 DNA | 说话节奏、句式、词汇、确定性、幽默 |
| 决策启发式 | 什么情况下推进、等待、拒绝、改变主意 |
| 压力反应 | 被催、被质疑、冲突、失败、焦虑时如何反应 |
| profile 专项 | 工作方法、关系互动、教学反馈或自我反思 |
| 反模式和边界 | 他讨厌什么、不会做什么、做不到什么 |
| 评估场景 | 可用于回放测试的历史场景和新问题 |

## 4. 问题生成策略

每轮按以下顺序选择问题：

1. 优先补最高权重 profile 维度。
2. 优先追问低置信度但影响大的结论。
3. 优先把抽象描述转成具体场景。
4. 优先请求原话、行为、时间和上下文。
5. 当用户给出强判断时，用反事实题校准。

## 5. 问题类型

### 场景回放题

用于从抽象印象变成证据。

示例：

- “你说他很直接。能回忆一次他直接否定别人方案的具体场景吗？”
- “当时是谁提出问题？他说了什么？对方怎么反应？”

### 对比题

用于捕捉特异性。

示例：

- “同样遇到拖延，他和你认识的其他人有什么不同？”
- “他在工作群和私聊里的表达有什么差异？”

### 反事实题

用于校准模型能否生成新判断。

示例：

- “如果有人绕过他直接找领导推进，他大概率会怎么反应？”
- “如果这个需求 impact 很高但实现很脏，他会接受吗？”

### 边界题

用于避免过度泛化。

示例：

- “什么问题上你不确定他会怎么想？”
- “有没有哪次他的反应和你对他的印象相反？”

### 语言指纹题

用于表达 DNA。

示例：

- “他常说的三句话是什么？”
- “他会用长段解释，还是一句话下结论？”

## 6. 约 20 轮后的提示

当轮次接近 20 且 coverage map 仍有关键缺口时，系统输出：

```text
目前已经能形成初版画像，但还有 2 个关键缺口：
1. 压力场景下的反应只有观察者印象，没有具体例子。
2. 表达风格有高频词，但缺少完整回复样本。

继续追问 3-5 轮可以明显提高这些部分的置信度。
如果你想停下来，我会把这两个部分标为低置信度并写入诚实边界。
```

## 7. 结束条件

满足以下条件之一即可结束访谈：

- 核心维度达到 medium 以上覆盖。
- profile 最高权重维度达到 high 或 medium。
- 用户明确停止。
- 用户提供更多语料替代继续访谈。

结束后输出访谈摘要：

- 已确认结论。
- 低置信度结论。
- 仍缺的信息。
- 建议补充材料。
